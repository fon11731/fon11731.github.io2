SEGA游戏服务器

WARNING

前言

目前关于 Sega 游戏服务器的搭建教程较为稀缺，且搭建过程需要一定技术基础。

HDD文件我也不会提供

开源的服务器项目主要有两个：

  • ARTEMIS
  • AquaDX

本文将先聚焦于 AquaDX 的搭建教程。

重要提示：本文分享的文件均不可外传！使用时请严格遵循各服务器的开源协议。

准备环境

ARTEMiS 服务端主要支持较旧的游戏版本，而 AquaDX 则更侧重支持较新的游戏版本。具体支持的游戏版本可参考官方说明：

  • ARTEMIS 支持游戏
  • AquaDX 支持游戏

环境要求对比

环境项 AquaDX 要求 ARTEMIS 要求
编译环境 Java Python
其他必要环境 Docker Docker

请根据选择的服务端提前准备好对应环境。

AquaDX 配置和部署

AquaDX 有两种部署方式：

  • Docker 直接部署（推荐）
  • 编译成 Jar 后部署

Docker 部署更快捷、自动化程度高且便于管理，本文以 Docker 部署为例进行演示。

备注：Java 编译过程较为繁琐且易出问题，因此优先推荐 Docker 方式 (x

步骤 1：获取 Docker 镜像

  • 官方镜像：Docker 官方镜像（默认反代 aquadx.net，不推荐新手使用）
  • 推荐镜像：自定义镜像（提取码：V8T0，解压密码：AquaDX）
  • 运行有问题的话看这个

步骤 2：配置服务器文件

下载并解压镜像后，需修改以下配置文件：

（1）编辑 compose.yml 文件

找到以下代码块：

services:
  http:
    image: caddy
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
    ports:
      - 8080:8080

修改 ports 中的端口号（如将 8080 改为其他端口），避免端口冲突。

（2）编辑 application.properties 文件

找到并修改以下项为你的局域网 IP（不可使用 127.0.0.1）：

# ALL.NET 服务器配置
allnet.server.host=192.168.0.1  # 局域网 IP
allnet.server.port=80           # ALL.NET 服务器端口

# 服务器重定向地址（需与 compose.yml 中的端口一致）
allnet.server.redirect=http://192.168.0.1:8080

# Web 端口配置
server.port=80                  # Web 访问端口

# 邮件服务器地址（一般无需使用，可留空）
aqua-net.email.webHost=192.168.0.1

若配置不生效，可尝试在 /AquaDX/config/application.properties 路径下再次按上述方法设置。

步骤 3：启动服务器

在服务器根目录打开终端，输入以下命令启动服务：

docker compose up

等待部署完成，当终端显示 ⭐️ ALL PERFECT ⭐️ 时，说明服务器成功运行。

步骤 4：验证服务器并注册账号

  1. 在浏览器中访问服务器地址：

    • 方式 1：局域网 IP + ALL.NET 端口（如 192.168.0.1:80）
    • 方式 2：allnet.server.redirect 配置地址（如 192.168.0.1:8080）

    若端口为 80，可直接输入 IP 访问（无需加端口）。

  2. 注册账号：

    • 输入任意邮箱和用户名，点击注册（无需验证邮箱，默认关闭验证）。
    • 使用注册的邮箱和密码登录即可进入服务器后台。

步骤 5：配置游戏客户端

根据获取的DNS和狗号信息，配置游戏客户端的 segatools.ini 文件，即可连接本地服务器。

关闭服务器

在终端中输入以下命令即可关闭服务器：

docker compose down

或者是在桌面直接 stop

注意事项

  • ARTEMIS 的官方教程更全面，若需搭建旧版本游戏服务器可参考其文档。
  • 请勿利用本地服务器逃避在线服的封号机制，此类行为无意义。
  • 禁止将此技术用于收费支持，该技术无特殊门槛，商业化行为不可取。
